# メンタルヘルスケアアプリ要件定義書

## 1. プロジェクト概要
LLMを使って使用者の悩みを聞いてくれるメンタルヘルスケアアプリ

## 2. 基本機能

### 2.1 対話機能
- **テキストチャット**: メイン機能
- **音声入力**: 音声からテキストに変換
- **音声読み上げ**: LLMの回答を音声で再生（ON/OFF切り替え可能）

### 2.2 LLMの役割
- 友人のような気軽な相談相手
- 慰めや共感に特化
- 例: 仕事でうまくいかなくて悩んでいるときに「あなたのせいじゃない」と慰めてくれる

### 2.3 セッション管理
- 継続的な対話履歴を保持
- 過去の相談内容を参照可能

## 3. ユーザー管理

### 3.1 認証方式
- **アカウント登録必須**
- **認証サービス**: Clerk

### 3.2 ターゲットユーザー
- **対象**: 社会人向け
- **利用シーン**: 職場、家庭などの悩み

### 3.3 プライバシー
- **完全匿名**: 個人情報の収集は最小限

## 4. データ・分析機能

### 4.1 履歴機能
- 過去の相談を振り返り可能
- **感情の変化可視化**: 毎日の気分を保存・グラフ化

### 4.2 データ保存
- 対話内容の暗号化保存
- バックアップ機能

## 5. UI/UX

### 5.1 デザインコンセプト
- **落ち着いた色合い**
- **親しみやすい**デザイン

### 5.2 操作性
- **シンプル重視**
- 直感的な操作

## 6. 技術仕様

### 6.1 プラットフォーム
- **モバイルアプリ**（iOS/Android）

### 6.2 技術スタック
- **フレームワーク**: Expo + React Native + TypeScript
- **状態管理**: Zustand
- **認証**: Clerk
- **データベース**: Supabase（PostgreSQL）
- **音声機能**: Expo AV + Speech-to-Text
- **LLM**: 複数API対応（OpenAI、Claude等）

## 7. ビジネスモデル

### 7.1 マネタイズ
**フリーミアムモデル**:
- **無料プラン**: 1日10回まで、基本機能のみ
- **プレミアムプラン**: 月額500-1000円
  - 無制限利用
  - 感情分析機能
  - 詳細履歴機能

## 8. 開発フェーズ

### 8.1 MVP機能
1. 基本チャット機能
2. ユーザー認証
3. 音声入力
4. シンプルな履歴保存

### 8.2 追加機能
1. 音声読み上げ
2. 感情分析・可視化
3. プレミアム機能
4. 利用制限システム

## 9. 技術アーキテクチャ

### 9.1 アプリ構成
```
Frontend (Expo/React Native)
├── 認証 (Clerk)
├── チャット画面
├── 音声機能
├── 履歴画面
└── 設定画面

Backend Services
├── Supabase (データベース)
├── LLM API (複数対応)
└── 音声処理サービス
```

### 9.2 データ設計
- **Users**: ユーザー情報（最小限）
- **Conversations**: 対話セッション
- **Messages**: メッセージ履歴
- **MoodLogs**: 感情ログ
- **Usage**: 利用回数管理
